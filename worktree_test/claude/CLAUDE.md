# Project Working Guide

## Purpose

이 파일은 Claude Code가 이 프로젝트에서 작업할 때 참고하는 프로젝트 로컬 작업 가이드다.
권한, 보안, 도구 실행 정책은 .claude/settings.json에서 관리한다.

## Work Principles

- 작업 전 관련 파일을 먼저 읽고 구조를 파악한다.
- 모르는 구조는 추측하지 않고 탐색한다.
- 기존 변수명, 함수명, 파일명은 불필요하게 변경하지 않는다.
- 요구사항과 직접 관련된 파일만 수정한다.
- 가능한 한 최소 변경으로 문제를 해결한다.
- 대규모 리팩토링은 명시 요청이 있을 때만 수행한다.

## Code Modification Principles

- 변경 이유를 설명할 수 있어야 한다.
- 기존 동작을 깨뜨릴 가능성이 있으면 먼저 리스크를 요약한다.
- 새 파일 생성보다 기존 구조 활용을 우선한다.
- 프로젝트 컨벤션이 확인되면 그 방식을 따른다.

## Verification Principles

- 가능한 경우 수정 전 현재 상태를 확인한다.
- 수정 후 가능한 검증 명령을 실행한다.
- 검증 명령을 알 수 없으면 임의로 만들지 않고, 확인 가능한 범위만 보고한다.
- 실패 로그가 있으면 원인을 요약하고 다음 수정 방향을 제시한다.

## Workflow

1. 요구사항 분석
2. 관련 파일 탐색
3. 변경 계획 수립
4. 코드 수정
5. 테스트 또는 실행 검증
6. 결과 요약

## Result Report Format

작업 완료 후 다음 항목을 요약한다.

- 변경 파일 목록
- 변경 이유
- 실행한 검증 명령
- 검증 결과
- 남은 리스크
